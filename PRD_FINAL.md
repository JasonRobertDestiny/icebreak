## 技术架构升级

### 当前技术栈（初赛）
```
Next.js 16 + DeepSeek API + LocalStorage
```

**问题**:
- 无用户系统
- 数据不持久
- 无法跨设备
- 技术力展示不足

### 新技术栈（决赛）

#### 1. 后端升级
```
Next.js 16 (前端)
  ↓
Supabase (数据库 + Auth + Storage)
  ├─ PostgreSQL: 用户数据、对话历史
  ├─ Auth: 手机号/微信登录
  └─ Storage: 头像、截图

DeepSeek API (AI)
  ├─ 话题生成
  ├─ 实时建议
  └─ 情感分析

Redis (缓存)
  ├─ 用户配额
  └─ 热点数据
```

#### 2. 用户系统
**功能**:
- 手机号注册/登录
- 微信快捷登录
- 个人资料
- 使用记录同步

**数据模型**:
```sql
-- 用户表
users (
  id, phone, wechat_openid,
  created_at, subscription_tier
)

-- 对话历史
conversations (
  id, user_id, target_name, target_interests,
  messages[], last_active, status
)

-- 关系管理
relationships (
  id, user_id, person_name, interests[],
  last_contact, next_reminder, health_score
)

-- 使用配额
usage_quotas (
  user_id, daily_generate, daily_realtime,
  subscription_tier
)
```

#### 3. 实时功能
**技术**: Server-Sent Events (SSE)

```typescript
// 实时建议流
async function* streamSuggestions(context: ConversationContext) {
  yield { type: 'analyzing', message: '正在分析对话...' };
  yield { type: 'suggestion', data: suggestion1, score: 85 };
  yield { type: 'suggestion', data: suggestion2, score: 78 };
  yield { type: 'complete' };
}
```
